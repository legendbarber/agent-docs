# Agent Docs — Lean Template

Codex와 Claude Code가 프로젝트를 이어서 작업할 때 필요한 컨텍스트만 유지하는 최소 문서 템플릿이다.

핵심 원칙은 다음과 같다.

- 코드, 테스트, 설정, Git 이력을 현재 상태의 원본으로 사용한다.
- 매 세션에 필요한 영구 지침만 `AGENTS.md`에 둔다.
- 변동이 잦은 상태와 설계 결정은 필요할 때만 읽는다.
- 작업 기록과 코드에서 쉽게 확인할 수 있는 내용은 문서화하지 않는다.

## 파일 구성

```text
AGENTS.md                    # Codex가 자동으로 읽는 짧은 프로젝트 지침
CLAUDE.md                    # Claude Code가 AGENTS.md를 가져오는 연결 파일
agent-docs/current-status.md # 진행 중인 목표·다음 작업·막힌 점
agent-docs/decisions.md      # 장기간 영향을 주는 비자명한 결정
```

`CLAUDE.md`는 `@AGENTS.md`만 포함한다. 두 도구용 지침을 따로 복사하지 않는다.

## 초기 작성

1. `AGENTS.md`의 핵심 컨텍스트를 실제 프로젝트 정보로 채운다.
2. 빌드·테스트·린트 명령은 직접 실행해 확인한 것만 적는다.
3. 코드만 읽어도 알 수 있는 폴더 목록이나 파일 설명은 적지 않는다.
4. 현재 이어서 진행할 작업이 있을 때만 `current-status.md`를 채운다.
5. 선택지가 있었고 향후 변경에 영향을 주는 결정만 `decisions.md`에 남긴다.

## 작업 중 읽기 절차

모든 문서를 미리 읽지 않는다.

- 일반 구현·수정: `AGENTS.md`와 관련 코드·테스트만 확인한다.
- 이전 작업 인계, 계획, 우선순위, 막힌 문제: `agent-docs/current-status.md`를 추가로 읽는다.
- 구조 변경, 기술 선택, 기존 결정의 이유 확인: `agent-docs/decisions.md`를 추가로 읽는다.

문서와 코드가 충돌하면 코드를 검증한 뒤 오래된 문서를 고친다.

## 작업 후 갱신 절차

기본값은 **문서를 갱신하지 않는 것**이다. 아래 조건에 해당할 때만 수정한다.

- `AGENTS.md`: 모든 세션에 필요한 영구 규칙, 검증된 명령, 핵심 제약이 바뀌었다.
- `current-status.md`: 다음 세션이 알아야 할 현재 목표, 다음 작업, 막힌 점이 바뀌었다.
- `decisions.md`: 의미 있는 대안 중 하나를 선택했고 그 선택이 이후 작업에 계속 영향을 준다.

완료한 작업의 시간순 기록, 사소한 행동, 변경 파일 목록, 코드와 중복되는 설명은 남기지 않는다. 완료 내역은 Git 커밋과 PR이 담당한다.

## 크기 기준

- `AGENTS.md`: 가능하면 100줄 이내, 최대 150줄
- `current-status.md`: 60줄 이내
- 결정 항목: 항목당 8줄 이내

한계를 넘기면 내용을 더 쪼개기 전에 오래된 정보, 중복, 코드에서 확인 가능한 사실부터 제거한다.
