## 작업 원칙

- 요청과 직접 관련된 코드, 테스트, 설정부터 확인한다.
- 코드, 테스트, 설정, Git 이력을 현재 상태의 원본으로 본다.
- `agent-docs/` 전체를 미리 읽지 않는다.
  - 이전 작업의 인계, 계획, 우선순위, 막힌 문제를 확인할 때만 `agent-docs/current-status.md`를 읽는다.
  - 구조 변경, 기술 선택, 기존 결정의 이유가 필요할 때만 `agent-docs/decisions.md`를 읽는다.
- 문서와 코드가 충돌하면 코드를 검증하고 오래된 문서를 수정한다.
- 요청 범위 밖의 리팩터링과 추측성 변경을 하지 않는다.
- 변경 후 가장 작은 관련 검증부터 실행하고, 실행하지 못한 검증은 결과에 명시한다.
- 공개 인터페이스, 데이터 형식, 의존성, 보안 경계를 바꿀 때는 요청 또는 명확한 필요가 있어야 한다.

## 문서 갱신 규칙

기본값은 문서를 갱신하지 않는 것이다. 다음 세션에 필요한 비자명하고 지속적인 정보가 생긴 경우에만 수정한다.

- `AGENTS.md`: 모든 작업에 적용되는 영구 규칙, 검증된 명령, 핵심 제약이 바뀐 경우
- `agent-docs/current-status.md`: 현재 목표, 다음 작업, 막힌 점이 다음 세션까지 이어지는 경우
- `agent-docs/decisions.md`: 의미 있는 대안 중 하나를 선택했고 그 결정이 이후 작업에 계속 영향을 주는 경우

다음 내용은 기록하지 않는다.

- 완료한 작업의 시간순 로그
- 사소한 도구 사용이나 내부 행동
- 변경 파일 목록
- 코드 검색으로 쉽게 확인할 수 있는 구조와 구현 세부사항
- 다른 문서에 이미 있는 내용

오래된 항목은 추가 설명을 덧붙이지 말고 교체하거나 제거한다. 하나의 사실은 한 곳에만 둔다.
